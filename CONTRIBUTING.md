# Contributing Guidelines

Thank you for considering contributing to this TDA Mapper repository!

## How to Contribute

- **Bug Reports & Feature Requests:** Please use GitHub Issues to report bugs or request features.
- **Pull Requests:**
  1. Fork the repository and create your branch from `main`.
  2. Ensure your code is well-documented and tested.
  3. Submit a pull request with a clear description of your changes.

## Code Style
- Follow PEP8 for Python code.
- Use clear, descriptive variable and function names.
- Add comments and docstrings where helpful.

## Notebooks
- Keep notebooks clean and well-commented.
- Remove unnecessary output before committing.

## Questions
For questions, open an issue or contact the repository maintainer.
